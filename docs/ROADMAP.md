# Roadmap — Embroidery Designer for iOS

Standalone native iOS app bringing Catrobat's embroidery functionality (Android "Embroidery Designer" flavor) to iOS: users program embroidery designs with visual blocks, watch the stitch preview on a stage, and export machine-readable Tajima DST files.

**Stack**: Swift 6, SwiftUI, min iOS 17. The engine (stitch domain, patterns, DST export, interpreter) lives in a platform-independent Swift Package (`EmbroideryEngine`) developed strictly test-first with `swift test`. License: AGPL-3.0 (Catrobat standard).

**Process**: TDD (failing tests first), user stories ≤ ~5 h, many small commits, one story per PR-sized unit of work. Stories for the *next* milestone are refined in detail; later milestones stay coarse until we get there.

**Engineering standards** (per the `swift-engineering` plugin skills — consult them while coding):
- **Swift Testing, not XCTest**: `@Test` / `#expect` / `#require`, parameterized tests via `@Test(arguments: zip(...))`. Tests run in parallel by default — they must not share mutable state or fixed file paths; serialize (`.serialized`) only as a last resort.
- **Swift 6 strict concurrency** enabled in the package from day one. Engine domain types are `Sendable` value types; the engine core stays synchronous and isolation-agnostic — actor/`@MainActor` decisions belong to the app layer (M3+), not the engine.
- Test fixtures load via `Bundle.module` (SPM resources); file-emitting tests write to unique temp URLs or assert on in-memory `Data`.
- App-layer side effects (file I/O, DST export/share, clock/sleep, UUID/date) sit behind small initializer-injected interfaces with deterministic test doubles (hand-rolled, no DI library — ADR-006). The engine takes no dependencies and performs no I/O.
- **Definition of done for every UI story (M3+)**: no hardcoded user-facing strings (String Catalog + translator comments; layout uses leading/trailing — Catrobat ships ~75 languages incl. RTL); loading/empty/error states shipped with the feature; animations respect Reduce Motion; basic VoiceOver pass (the stage canvas gets a summarizing label: stitch count, colors, size in mm); touch targets ≥ 44 pt; Dynamic Type to AX1 without truncation.

## Milestones

### M1 — Engine core (E1 + E2)
**Status**: Done — 2026-07-16. All ten stories (US-101–US-110) merged; exit criterion met (US-106 golden-file tests green in CI; generated DST verified in Ink/Stitch, see workflow journal 2026-07-09).

A pure Swift package that turns a programmatic sequence of needle movements into a **valid DST file**, byte-verified against the reference fixture from Catty (`CattyTests/Resources/EmbroideryReference/stitch.dst`) and the semantics of Catroid's `org.catrobat.catroid.embroidery`. No UI. CI runs `swift test` on every push.

**Exit criterion**: a DST file generated by our engine loads correctly in an embroidery viewer and matches golden-file expectations (header, stitch records, jumps, color changes, terminator).

Detailed stories: [`user-stories/milestone-1/`](user-stories/milestone-1/)

### M2 — Interpreter MVP (E3)
Program model (project → scene → object → scripts → bricks) plus a minimal interpreter executing headlessly: when-started script, move/turn/goto, repeat/wait, the eight embroidery bricks, and a formula subset (literals, arithmetic, variables). Executing a program yields an `EmbroideryStream`. Scripts are flat brick lists with paired control bricks; the move-a-pair-as-a-unit invariant is model logic with tests (ADR-008). The virtual needle operates in the ADR-007 stage space. The interpreter ships as a **sibling target in the same package** (`ProgramModel`/interpreter depending on the engine target, never the reverse) so M4's editor and M5's persistence can import the program model without dragging DST I/O along.

**Exit criterion**: a hardcoded "stitch a square/star" program produces the expected stitch stream in unit tests — still no UI — and the interpreter is **incrementally consumable**: execution advances step-by-step, emits ordered needle/stitch/color events, is deterministic (golden stitch-stream tests), and takes an injected clock for time-based bricks. Unit test: consuming events one at a time yields the same stream as batch execution.

### M3 — Walking skeleton app (E4 + E7, thin E6)
Minimal SwiftUI app wired end-to-end: pick a bundled sample program → run it with live stage preview (needle + stitches) → export the DST via share sheet. Proves the full thread through all layers. App-layer state uses iOS 17 `@Observable` view models on `@MainActor` (ADR-006); interpreter output reaches the preview via `AsyncStream` from the cancellable run `Task`, **batched before mutating observable state** (never one UI update per stitch); the run lifecycle is a small state enum (idle/running/finished/failed).

**In scope**: fit-to-screen stage (Canvas renderer per ADR-009), play/stop, needle indicator, run states, share-sheet export with an **exported UTType for `.dst`** (Info.plist declaration + `Transferable`/`FileRepresentation` — no system type exists) and design-name validation (≤15 ASCII chars, shown as a live counter, not a post-hoc error); String Catalog from the first UI commit; size-class-adaptive navigation skeleton (ADR-010); ≥2 bundled samples that are visually appealing designs exercising zigzag/triple/color-change — not test shapes. **Out**: scrubbing, speed control, editing (pinch-zoom/pan only if the ADR-009 transform work makes it nearly free).

**Exit criteria**: end-to-end sample→preview→DST works; a synthetic 50k-stitch design animates at 60 fps on an A15-class device; zoom/pan transform math is unit-tested.

### M4 — Block editor (E5)
Create and edit programs in the app: brick palette as a bottom sheet with detents (tap-to-add, matching Catroid's flow — no free-form drag-from-palette), reorder via `List` + `.onMove` (flat list per ADR-008; custom accessibility actions as fallback; each brick is a single VoiceOver element with parameter values in its label; brick layout reflows under Dynamic Type). Parameter editing is **type-specific** (number pad, color picker / curated thread palette, variable menu, minimal arithmetic entry) — a full Catroid-style formula-editor keyboard is explicitly deferred (M6/stretch; don't build editor surface for functions the M2 interpreter doesn't have).

Editor mutations are a pure `EditAction` set applied to the value-type program in the SPM package (UI-free, developed test-first with full-`Program` equality assertions — ADR-006); the first M4 story is this headless EditorCore including the **undo/redo snapshot stack** (bounded, coalesced per gesture, `UndoManager` bridging for shake/Cmd-Z). All editor mutations are undoable from day one. One thin persistence slice lands here too: **implicit autosave** of the single working program to Documents as JSON (no project list or versioning UI — that stays M5), so the editor is usable across restarts and testable with real users.

**Checkpoint after M4**: informal usability session with 3–5 target-age users via TestFlight; findings triaged into the M5/M6 backlogs.

### M5 — Project management (E6)
Project list (create/rename/duplicate/delete — rename validates the 15-char DST name rule inline), JSON (Codable) on-disk format with versioning (autosave itself lands in M4), bundled sample designs.

### M6 — Brick parity & polish (E8)
Full parity with the Android flavor's embroidery brick set and behaviors — the concrete parity target is the 8 embroidery bricks plus Catroid's embroidery-category motion bricks (PlaceAt, SetX/SetY, MoveNSteps, TurnLeft/TurnRight, PointInDirection, Arc, GoThrough) and the control/data subset; list it as a checklist early so M6 isn't discovering scope. Polish is an **audit**, not a build-out (the M3+ definition of done keeps states/accessibility/strings continuous): pseudo-localization + RTL pass, sparse haptic map (success on export/run-complete, selection tick on brick add/reorder-drop, error on export failure — nothing else), app icon, App Store prep (age rating, no-tracking declaration per ADR-011) — ready for sign-off and transfer to the Catrobat org.

### Stretch (time permitting)
Freehand path drawing converted to stitch patterns; `.catrobat` import for Share-platform interop; SVG plotting (Catty precedent).
